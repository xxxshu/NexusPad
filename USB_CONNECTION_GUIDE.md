# NexusPad USB 连接问题解决指南

## 问题描述

根据诊断信息，发现了两个主要的 USB 连接问题：

### 1. "仅充电"模式问题
- **设备状态**: `18D1:2D01 (无法打开) [AOA]`
- **问题**: 设备已进入 AOA 模式，但 Windows 缺少正确的 WinUSB 驱动
- **表现**: 设备被检测到，但无法打开（权限不足）

### 2. "传输文件/Android Auto"模式问题  
- **设备状态**: `22D9:2765 (无法打开) [Android] → 打开失败: Access denied (insufficient permissions)`
- **问题**: 设备保持在 Android 模式，权限被系统/MTP/ADB 驱动占用
- **表现**: 权限不足，无法直接访问设备

## 根本原因

Windows USB 连接有两种模式：
1. **AOA (Android Open Accessory) 模式**: 需要 WinUSB 驱动
2. **Android 模式**: 被系统驱动占用，无法直接访问

## 解决方案

### 方案 A: 安装 WinUSB 驱动 (针对 AOA 模式)

如果看到 `18D1:2D01` 设备：

1. **下载 Zadig 工具**: https://zadig.akeo.ie/
2. **运行 Zadig**:
   - 点击 Options → List All Devices
   - 在列表中找到 Android 或 NexusPad 设备
3. **安装驱动**:
   - 驱动选择: `WinUSB (libusb-win32)`
   - 点击 Replace Driver
4. **重新连接**:
   - 拔插 USB 线
   - 手机切换到「仅充电」模式
   - 重试连接

### 方案 B: 切换 USB 模式 (针对 Android 模式)

如果看到 `22D9:2765` 设备：

1. **在手机上切换模式**:
   - 下拉通知栏
   - 找到「USB 用于」选项
   - 尝试不同模式:
     - **仅充电** (推荐先试)
     - **文件传输**
     - **Android Auto**
2. **重试连接**:
   - 切换模式后等待 5-10 秒
   - 点击 NexusPad 的 START 按钮

## 使用步骤

1. **打开 NexusPad 桌面端**
2. **进入设置 → 通用 → USB 连接**
3. **使用调试工具**:
   - 点击 **驱动检测** 查看当前状态
   - 点击 **详细调试** 生成日志
   - 点击 **驱动修复指导** 获取解决方案
4. **根据指导操作**:
   - 按照指导安装驱动或切换模式
   - 重试 USB 连接

## 关键提示

1. **模式选择重要**:
   - 不同手机/系统可能需要不同模式
   - 建议逐个尝试所有可用的 USB 模式
   - 「仅充电」模式通常最容易成功

2. **驱动修复**:
   - WinUSB 驱动仅针对 AOA 模式
   - 安装后可能需要重启手机
   - 某些设备需要特定版本的驱动

3. **手机端要求**:
   - 确保 NexusPad app 已启动
   - 选择 USB 连接方式
   - 点击 START 按钮
   - 允许弹出的权限请求

4. **备用方案**:
   - 如果 USB 连接持续失败
   - 可以暂时使用 Wi-Fi 或蓝牙连接
   - USB 主要用于低延迟场景（游戏手柄/陀螺仪）

## 常见问题

### Q: 为什么需要 WinUSB 驱动？
A: Windows 默认使用 MTP/ADB 驱动访问 Android 设备，这些驱动独占设备访问权限。WinUSB 是标准的 USB 驱动，允许应用程序直接访问 USB 设备。

### Q: 安装驱动安全吗？
A: Zadig 是开源工具，WinUSB 是微软的标准 USB 驱动。安装后可以随时在设备管理器中恢复原始驱动。

### Q: 为什么不同模式表现不同？
A: 手机在不同模式下使用不同的 USB 配置和驱动：
- **仅充电**: 最小化功能，容易切换到 AOA
- **文件传输**: 启用 MTP 驱动，占用设备
- **Android Auto**: 启用特定功能，可能干扰 AOA

## 性能考虑

- **AOA 连接** 提供最低延迟（<2ms）
- **适合** 游戏手柄、陀螺仪等低延迟场景
- **不适合** 文件传输等大数据量场景
