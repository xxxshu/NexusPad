## 一、Apple MacBook 触控板技术剖析

### 1.1 Force Touch 与 Taptic Engine

Apple 在 2015 年推出的 Force Touch 触控板取消了传统的机械铰链（"diving board" 结构），取而代之的是一个**几乎不移动**（仅约 0.1mm）的固态表面。点击手感完全由 **Taptic Engine**（线性谐振致动器，LRA）模拟产生。

**LRA 工作原理**：
- 内部由**弹簧悬挂的质量块（磁铁）** + **电磁线圈**组成
- 线圈驱动质量块沿直线快速往复运动，产生触觉振动
- 工作频率约 **170Hz ~ 235Hz**（不同代际有差异）
- 仅需**一个振动周期**即可产生清脆的点击反馈
- 可产生**两级力反馈**：轻触（普通点击）和深压（Force Click）

**力传感器阵列**：
- 触控板下方有**四个应变片力传感器**（角落各一个），通过柔性印刷电路连接到控制板
- 四点布局可**三角定位**按压位置和力的大小
- 配合力数据，系统在合适时机激活 Taptic Engine

```
┌─────────────────────────┐
│ [力传感器]           [力传感器] │
│                         │
│   电容式触摸传感器阵列    │
│   (发射电极 × 接收电极)  │
│                         │
│ [力传感器]  [Taptic] [力传感器] │
│           [Engine]        │
└─────────────────────────┘
```

**触控板分层结构**（从上到下）：
1. **玻璃表面** + 导电涂层
2. **电容传感器层**（电极网格，用于位置追踪）
3. **力传感器层**（四个应变片，仅 Force Touch 型号）
4. **Taptic Engine**（LRA 线性致动器，仅 Force Touch 型号）
5. **控制器 PCB**（BCM5974 芯片）

### 1.2 多点触控手势识别

**触摸数据流水线**（从手指到手势）：

```
硬件层 (BCM5974 控制器)
  ↓ HID 报告：每指位置、大小、力度
内核驱动层 (AppleUSBMultitouchDriver / IOHIDFamily)
  ↓ 防误触 + 基础滤波
WindowServer / SkyLight 框架
  ↓ 命中测试 (Hit Testing)
NSEvent 传递
  ↓ 先到手势识别器，再到视图
NSGestureRecognizer 处理
  ↓ 分析 NSTouch 流，确定手势类型
应用程序
```

**NSTouch 对象属性**：
| 属性 | 说明 |
|------|------|
| `identity` | 唯一标识符，用于跨事件追踪同一手指 |
| `phase` | `.began` / `.moved` / `.stationary` / `.ended` / `.cancelled` |
| `normalizedPosition` | 相对于触控板表面的归一化坐标 |
| `isResting` | 是否来自搁置的手掌（防误触用） |

> macOS 触控板坐标空间为 **0 ~ 65535**（16 位），提供极高精度。

**macOS 内置手势识别器**：
| 识别器 | 手势 |
|--------|------|
| `NSMagnificationGestureRecognizer` | 双指捏合/缩放 |
| `NSRotationGestureRecognizer` | 双指旋转 |
| `NSPanGestureRecognizer` | 拖拽/平移 |
| `NSClickGestureRecognizer` | 点击（离散手势） |
| `NSPressGestureRecognizer` | 长按 |

**系统级三指/四指手势**：
| 手势 | 默认动作 |
|------|----------|
| 三指上滑 | Mission Control |
| 三指下滑 | App Exposé |
| 三指左右滑 | 切换桌面空间 |
| 四指上滑 | Mission Control |
| 四指下滑 | 显示桌面 / Launchpad |

### 1.3 手势状态机

Apple 的 `NSGestureRecognizer` 使用标准的**有限状态机**：

| 状态 | 值 | 说明 |
|------|----|------|
| `possible` | 0 | 初始状态；可能被识别 |
| `began` | 1 | 触摸被识别为手势开始 |
| `changed` | 2 | 手势持续变化 |
| `ended` | 3 | 手势完成 |
| `cancelled` | 4 | 手势被取消 |
| `failed` | 5 | 未识别 |

**离散手势**（tap、click、swipe）：
```
possible ──→ ended (识别成功)
possible ──→ failed
```

**连续手势**（pinch、pan、rotation、scroll）：
```
possible ──→ began ──→ changed ──→ ... ──→ changed ──→ ended
                     └──→ cancelled
```

**触摸传递顺序**：
1. 命中测试确定接收触摸的视图
2. 触摸先传递给视图及其父视图上的**手势识别器**（先于视图本身）
3. 识别器的 `touchesBegan` → `touchesMoved` → `touchesEnded` 被调用
4. `canPrevent` / `canBePrevented` 控制识别器之间的互斥关系
5. `require(toFail:)` 强制一个识别器等待另一个失败

### 1.4 加速度曲线

macOS 指针加速实现在 **IOHIDFamily** 内核扩展中（开源），关键函数为 `__IOHIDApplyAcceleration`。

**实现方式：分段线性查找表**

Apple **不使用**简单的多项式或指数函数，而是使用一个包含约 **50-60 个 (输入速度, 输出增益) 对**的**分段线性查找表**，表项之间使用**线性插值**。

**处理流程**：
1. 系统在短时间间隔内测量触控板/鼠标速度（counts/ms）
2. 在加速表中查找速度对应的增益倍数
3. 将增益乘以原始位移得到最终输出

**曲线形状**（凹形）：

```
输出增益
  │
  │                              ╱
  │                           ╱
  │                        ╱
  │                    ╱
  │              ╱╱
  │          ╱╱
  │      ╱╱        ← 中速区：逐渐加速
  │   ╱╱
  │ ╱╱             ← 低速区：接近 1:1，精确控制
  │╱
  └──────────────────────────→ 输入速度
```

**固定精度表示**（内核/用户空间共享）：
```c
#define kIOHIDPointerScaleResolution  10
#define kIOHIDPointerScaleFixedOne    (1 << 10)  // = 1024
```
使用 **6.10 定点数**（10 位小数位）。

**灵敏度缩放**：系统偏好设置中的"跟踪速度"滑块（`com.apple.trackpad.scaling`，范围 0~3）会整体缩放加速曲线。值越大，每个速度点的增益越高。

**关键逆向工程资源**：
- [Felix Geisendörfer 交互式加速计算器](https://www.felixge.de/mouse-acceleration/)
- [Alin Panaitiu 深入分析 IOHIDFamily 源码](https://notes.alinpanaitiu.com/mouse-acceleration-macos-part1)
- [macOS 加速曲线逆向 Python 实现](https://gist.github.com/stephen-huan/2c17f1a84eb8c979427f5e1798701995)
- [IOHIDPointerScrollMapper.cpp 源码](https://github.com/apple-oss-distributions/IOHIDFamily/blob/master/IOHIDFamily/IOHIDPointerScrollMapper.cpp)

### 1.5 惯性滚动与物理模型

当手指离开触控板时，macOS 继续以递减的速度滚动（惯性滚动）。

**指数衰减模型**：

```
v(t) = v₀ · e^(-k·t)           // 速度随时间指数衰减
x(t) = x₀ + (v₀/k) · (1 - e^(-k·t))  // 位移收敛
```

**UIKit 已知常量**（概念上适用于 macOS）：
| 常量 | 值 | 说明 |
|------|----|------|
| `UIScrollViewDecelerationRateNormal` | 0.998 | 每帧速度衰减因子 |
| `UIScrollViewDecelerationRateFast` | 0.990 | 快速滚动衰减因子 |

**逐帧更新公式**：
```javascript
// 每帧执行
velocity *= decelerationRate;   // 速度衰减
position += velocity;           // 更新位置
// 当 |velocity| < 0.5 时停止
```

**时间标准化版本**（适配不同刷新率）：
```javascript
velocity = velocity * Math.pow(decelerationRate, deltaTime);
position += velocity * deltaTime;
```

> macOS 内部使用 `_NSScrollingMomentumCalculator` 类处理减速计算，配合 `CVDisplayLink` 以显示器刷新率驱动动画。

### 1.6 橡皮筋效果（Overscroll Bounce）

当滚动超出内容边界时，macOS 使用**渐近函数**模拟橡皮筋拉伸效果：

```
offset = (1 - 1/(x·c/d + 1)) · d
```

其中：
- `x` = 超出边界的距离
- `c` = 系数（通常约 0.55，控制阻力感）
- `d` = 最大视觉位移（通常等于视口尺寸）

**特性**：越拉越难拉，趋近但不超过 `d`，产生弹性拉伸感。

**回弹动画**使用阻尼谐振子（弹簧物理）：
```
x(t) = A · e^(-γ·t) · cos(ω·t + φ)
```

UIKit 的 UIScrollView 大约使用弹簧常数 `k ≈ 167`，阻尼比 `ζ ≈ 0.5`。

**简化版橡皮筋函数**（Web/移动端常用）：
```javascript
function rubberBand(offset, dimension, constant = 0.55) {
    return (1.0 - (1.0 / ((Math.abs(offset) * constant / dimension) + 1.0)))
           * dimension * Math.sign(offset);
}
```

### 1.7 防误触（Palm Rejection）

Apple 的防误触是多层级系统，结合硬件与软件：

**触摸尺寸和形状分析**：
- 电容传感器测量每个触摸点的**接触面积**
- 指尖产生小型聚焦接触点（~5-8mm 直径）
- 手掌产生大型不规则接触区域（~20-40mm+）

**上下文感知拒绝**：
- **键盘活动感知**：检测是否近期有按键输入（打字上下文）。按键后短时间内出现在"掌区域"的触摸被抑制
- **触摸速度与运动模式**：手掌趋向缓慢移动或静止，手指更有目的性
- **多点关联**：大面积接触与活跃手指运动同时出现时，大面积被归类为手掌
- **边缘拒绝**：在触控板边缘区域应用更严格的拒绝标准（手掌自然搁置位置）

**Force Touch 增强**：四力传感器的压力数据提供额外维度来区分有意与无意接触。`NSTouch` 的 `isResting` 属性标记手掌触摸。

---

### 1.8 触控板传感器与表面技术

**电容传感器阵列**：
- 使用**互电容**感应方式
- 两层电极网格：**发射电极**（一个轴）× **接收电极**（另一个轴）
- 每个交叉点独立测量，形成触摸强度的**热力图**
- 支持多点同时触摸检测

**BCM5974 控制器**：
- 由 Broadcom BCM5974 多点触控控制器驱动
- 坐标空间为 **0 ~ 65535**（16 位），X/Y 轴均有
- 物理传感器分辨率较低，通过插值达到 16 位精度
- 每个型号可配置**模糊度**（噪声过滤）和**分辨率**参数

> [Linux BCM5974 驱动源码](https://github.com/torvalds/linux/blob/master/drivers/input/mouse/bcm5974.c) 详细记录了这些参数。

### 1.9 macOS 触控板配置参数

**GUI 配置**：系统设置 > 触控板（三个标签页：光标与点按、滚动与缩放、更多手势）

**命令行配置**：
```bash
# 跟踪速度 (0 = 最慢, 3 = 最快)

# 读取所有触控板设置
defaults read com.apple.AppleMultitouchTrackpad
```

## 二、Windows / Linux 触控板实现对比

### 2.1 Windows Precision Touchpad (PTP)

**驱动栈架构**：
```
用户模式手势引擎 (Windows.UI.Input.Core)
  ↓
HID 类驱动 (hidclass.sys)
  ↓ 解析 HID Report Descriptors
HID 微型端口传输 (usbhid.sys / HIDI2C.sys)
```

**PTP HID 协议**：使用 Digitizer usage page (`0x0D`)，usage `0x05`（Touch Pad）。每个报告包含：
| 字段 | Usage | 说明 |
|------|-------|------|
| Tip Switch | `0x42` | 手指触摸状态 |
| Confidence | `0x47` | 防误触标志 |
| Contact ID | `0x51` | 每指独立追踪 |
| X/Y | `0x30`/`0x31` | 坐标 |
| Width/Height | `0x48`/`0x49` | 接触几何 |
| Pressure | `0x30` | 压力（可选） |

**PTP 加速**：使用**平滑样条曲线**（非旧版离散阶梯）。报告间隔 8-12ms。

**旧版 Windows 鼠标加速**（"增强指针精确度"）：
```
if speed > MouseThreshold2 (10):
    multiplier = 4
elif speed > MouseThreshold1 (6):
    multiplier = 2
else:
    multiplier = 1
```

**DirectManipulation API** 提供惯性处理：
- 减速模型：`v(t) = v₀ · e^(-k·t)`

### 2.2 Linux libinput

**源码**：[libinput/src/filter.c](https://gitlab.freedesktop.org/libinput/libinput/-/blob/master/src/filter.c) | [filter-touchpad.c](https://gitlab.freedesktop.org/libinput/libinput/-/blob/main/src/filter-touchpad.c)

**架构**：`struct pointer_accelerator` 持有加速状态。配置文件实现 `struct pointer_filter` 接口。

**速度追踪**：使用**指数衰减平均**——维护一个近期运动样本历史，按指数衰减加权，较新样本权重更高，产生平滑的速度估计。

**加速配置文件**：
| 配置 | 说明 |
|------|------|
| Flat | 无加速，恒定 `accel_factor = 1.0` |
| Adaptive（触控板默认） | 分段多项式曲线 |
| Trackpoint | 专为小红点调整的多项式 |

**自适应配置文件公式**（三次多项式）：
```
accel_factor = a₀ + a₁·v + a₂·v² + a₃·v³
output_dx = input_dx · accel_factor(velocity)
```

曲线分为多段，段间保持 C1 或 C2 连续。低速区接近 1:1（精确控制），高速区多项式上升（快速跨越屏幕）。

**防误触**（libinput 实现）：
- **触摸大小**：`ABS_MT_TOUCH_MAJOR` 超过阈值（~12-14mm）→ 判定为手掌
- **边缘区域**：触控板边缘 5-8mm 内的触摸激进判定为手掌
- **静止超时**：触摸静止超过 ~300ms 可能被重分类为手掌
- **运动分析**：手掌移动速度特征与手指不同

## 三、核心算法详解

### 3.1 指针加速算法

#### macOS 方式：查找表 + 线性插值
（详见 [1.4 加速度曲线](#14-加速度曲线)）

#### Windows PTP 方式：样条曲线
（详见 [2.1 Windows Precision Touchpad](#21-windows-precision-touchpad-ptp)）

#### libinput 方式：分段三次多项式
（详见 [2.2 Linux libinput](#22-linux-libinput)）

#### Raw Accel（开源 Windows 内核驱动）

GitHub: [a1xd/rawaccel](https://github.com/a1xd/rawaccel) / [RawAccelOfficial/rawaccel](https://github.com/RawAccelOfficial/rawaccel)

提供多种曲线类型：

**Natural**（最流行）：
```
output = input × (1 + accel × (1 - e^(-(|v| / offset)^power)))
```
参数：`accel`、`offset`、`power`。产生平滑、有界的加速效果。

**Power（经典）**：
```
output = input × (accel × |v|^(exp - 1))
```

**Linear**：
```
output = input × (accel × |v| + 1)
```

**通用参数**：sensitivity（基础倍率）、offset（加速启动速度阈值）、cap（最大输出灵敏度）。

#### 实用的 Web 触控板加速函数

```typescript
function applyAcceleration(dx: number, dy: number): [number, number] {
  const speed = Math.sqrt(dx * dx + dy * dy);
  const dt = Date.now() - lastMoveTime;
  lastMoveTime = Date.now();

  // 速度（像素/毫秒）
  const velocity = speed / Math.max(dt, 1);

  let multiplier: number;
  if (velocity < 0.5) {
    // 慢速移动：精确模式（1:1）
    multiplier = 1.0;
  } else if (velocity < 2.0) {
    // 中速移动：适度加速
    multiplier = 1.0 + (velocity - 0.5) * 0.8;  // 线性斜坡 1.0 → 2.2
  } else {
    // 快速移动：强力加速
    multiplier = 2.2 + Math.min((velocity - 2.0) * 0.5, 3.0);  // 上限 5.2x
  }

  return [dx * multiplier, dy * multiplier];
}
```

### 3.2 触摸点预处理与滤波

#### One Euro Filter（业界最佳触摸/指针平滑算法）

论文：Casiez, G., Roussel, N., & Vogel, D. (2012). "1 Euro Filter: A Simple Speed-based Low-pass Filter for Noisy Input in Interactive Systems." CHI 2012。

官网：[cristal.univ-lille.fr/~casiez/1euro/](http://cristal.univ-lille.fr/~casiez/1euro/)

**算法核心**：

```javascript
// 1. 计算速度（导数）
dx = (x - prev_x) / dt;
smoothed_dx = alpha(dcutoff) * dx + (1 - alpha(dcutoff)) * prev_smoothed_dx;

// 2. 根据速度计算自适应截止频率
cutoff = min_cutoff + beta * Math.abs(smoothed_dx);

// 3. 以自适应截止频率应用低通滤波
filtered_x = alpha(cutoff) * x + (1 - alpha(cutoff)) * prev_filtered_x;

// alpha 函数：将截止频率转换为平滑因子
function alpha(cutoff) {
  return 1.0 / (1.0 + (1.0 / (2.0 * Math.PI * cutoff * dt)));
}
```

**参数说明**：
| 参数 | 典型值 | 说明 |
|------|--------|------|
| `min_cutoff` | 1.0 | 越低 = 静止时越平滑（但延迟更大） |
| `beta` | 0.007 | 越高 = 快速移动时延迟越小 |
| `dcutoff` | 1.0 | 通常保持不变 |

**关键行为**：
- **静止/慢速** → 截止频率接近 `min_cutoff` → 强平滑（消除抖动）
- **快速移动** → 截止频率随速度增加 → 弱平滑（保持响应性）

这种自适应特性使其**远优于**固定低通滤波器。

#### 指数移动平均（EMA）

更简单的替代方案：
```
smoothed = alpha × new_input + (1 - alpha) × previous_smoothed
```
- alpha 0.1~0.3：平滑但延迟大
- alpha 0.7~0.9：响应快但不够平滑
- **自适应 alpha**：根据速度动态调整——慢速时更平滑，快速滑动时更响应

#### 卡尔曼滤波器

1. **预测步**：`x_predicted = x_prev + v_prev × dt`
2. **更新步**：`x_estimated = x_predicted + K × (z_measured - x_predicted)`
3. 卡尔曼增益 `K` 由过程噪声 `Q` 和测量噪声 `R` 计算得出

#### 电容传感器网格处理流水线

触控板控制器 IC（Synaptics、Elan、Alps Alpine）内部：

```
原始传感器扫描（电容测量网格/热力图）
  ↓ 噪声滤波（移动平均/中值/低通 FIR/IIR 滤波器，去除 EMI）
  ↓ 插值（相邻传感器节点间双线性插值，获得亚传感器精度）
  ↓ 峰值检测（识别触摸 blob 中心）
  ↓ 质心计算（加权质心算法）
```

**加权质心算法**：
```
X = Σ(reading_i × x_i) / Σ(reading_i)
Y = Σ(reading_i × y_i) / Σ(reading_i)
```

变体包括高斯加权质心和插值质心，可实现亚传感器节点精度。

### 3.3 手势识别算法

#### 判断不同手势的阈值参数

| 手势 | 距离阈值 | 速度阈值 | 时间窗口 |
|------|----------|----------|----------|
| 轻触 (Tap) | < 10-20px | N/A | < 200-300ms |
| 滑动 (Swipe) | > 50-100px | > 300-1000 px/s | < 300-500ms |
| 捏合 (Pinch) | > 10px 变化量 | N/A | 连续 |
| 旋转 (Rotation) | > 2° 变化量 | N/A | 连续 |

#### 捏合缩放

监测两点之间的**欧几里得距离**变化：
```javascript
scale = currentDistance / initialDistance;
// scale > 1 = 放大, scale < 1 = 缩小

function getDistance(t1, t2) {
  const dx = t1.clientX - t2.clientX;
  const dy = t1.clientY - t2.clientY;
  return Math.sqrt(dx * dx + dy * dy);
}
```

#### 旋转

监测两点相对于质心的**角度变化**：
```javascript
angle = Math.atan2(y2 - y1, x2 - x1);
rotationDelta = currentAngle - initialAngle;
```

#### 区分捏合与旋转

同时计算两个指标，以先超过阈值的为主判定：
- 距离变化大 + 角度变化小 → **捏合**
- 角度变化大 + 距离变化小 → **旋转**
- 混合 → **捏合+旋转组合**

#### 多指追踪

每根手指通过**Contact ID**（跨 HID 报告帧持续存在）追踪。手指抬起时 Contact ID 从报告中消失。手势引擎需要：
1. 维护活跃 Contact ID 集合
2. 独立计算每指速度
3. 计算指间指标（距离、角度）用于多指手势
4. 处理手势中途手指增减（如从单指开始，中途加入第二指开始捏合）

### 3.5 精确模式与慢速移动处理

#### 亚像素精度

触控板数据以原始整数坐标到达（通常远高于屏幕像素分辨率）：

```javascript
// 浮点累积法
accumulated_x += raw_dx * accelerationFactor;
accumulated_y += raw_dy * accelerationFactor;

// 仅当累积值跨越整像素边界时才移动光标
pixel_dx = Math.trunc(accumulated_x);
accumulated_x -= pixel_dx;  // 保留小数余量给下一帧
```

#### 初始移动死区

**硬死区**：必须超过固定阈值才开始移动：
```javascript
distance = Math.sqrt(accumulated_x² + accumulated_y²);
if (distance > deadZoneThreshold) {
  moveCursor(accumulated_x, accumulated_y);
  accumulated_x = 0;
  accumulated_y = 0;
}
```

**软死区**：使用指数/逻辑曲线平滑过渡：
```javascript
output = Math.sign(input) * (Math.abs(input) / (Math.abs(input) + deadZoneK));
```

**丢弃移动死区**：越过阈值后只应用超出部分（不累积），防止退出死区时的"跳跃"。

### 3.6 防误触算法

#### libinput 的多重启发式方法

| 启发式 | 描述 |
|--------|------|
| 触摸大小 | `ABS_MT_TOUCH_MAJOR` > 阈值（~12-14mm）→ 手掌 |
| 边缘区域 | 触控板边缘 5-8mm 内的触摸激进判定为手掌 |
| 静止超时 | 触摸静止 ~300ms 可能被重分类为手掌 |
| 运动分析 | 手掌移动速度特征与手指不同 |
| 接触形状 | 手掌：长椭圆或不规则；指尖：近圆形 |

**接触形状分析**：
- **纵横比**：`touch_major / touch_minor`（手掌更大）
- **面积**：`π × touch_major × touch_minor / 4`

#### 机器学习方法

现代实现（Apple、Synaptics、Elan）使用 ML 模型，特征包括：
- 接触面积和形状描述符
- 速度和加速度分布
- 多点关联（手掌出现在活跃手指旁边）
- 时间模式（接触如何随时间演变）
- 相对触控板边缘的位置
- 压力梯度分布（手掌压力更分散）

> 模型必须在嵌入式触控板控制器上实时运行，因此使用轻量级模型（决策树、小型神经网络或 SVM）。

## 五、Web 端触控板最佳实践

### 5.1 触摸事件处理

```typescript
// 关键：使用 { passive: false } 才能 preventDefault()
element.addEventListener('touchstart', onTouchStart, { passive: false });
element.addEventListener('touchmove', onTouchMove, { passive: false });
element.addEventListener('touchend', onTouchEnd);
element.addEventListener('touchcancel', onTouchCancel);

function onTouchStart(e: TouchEvent) {
  e.preventDefault(); // 关键：阻止浏览器默认滚动/缩放
  if (e.touches.length === 1) {
    // 单指：准备光标移动
    startX = lastX = e.touches[0].clientX;
    startY = lastY = e.touches[0].clientY;
    tapStartTime = Date.now();
    gestureState = 'tracking';
  } else if (e.touches.length === 2) {
    // 双指：准备滚动或捏合
    initialPinchDistance = getDistance(e.touches[0], e.touches[1]);
    gestureState = 'multi-touch';
  }
}
```

**核心原则**：触控板风格意味着触摸表面就是鼠标，始终发送**增量**（而非绝对位置）。手机屏幕是相对定位设备。

### 5.2 手势状态机设计

```
IDLE ──→ (touchstart, 1 指) ──→ SINGLE_FINGER_TRACKING
IDLE ──→ (touchstart, 2 指) ──→ MULTI_TOUCH

SINGLE_FINGER_TRACKING ──→ (touchmove):
  移动 < 阈值 → TAP_CANDIDATE（轻触候选）
  移动 > 阈值 → TRACKING（光标移动）

SINGLE_FINGER_TRACKING ──→ (touchend):
  TAP_CANDIDATE → 检查时序/距离 → 单击或双击

MULTI_TOUCH ──→ (touchmove):
  指间距变化 → PINCH
  质心移动 + 指间距稳定 → SCROLL

MULTI_TOUCH ──→ (touchend) → IDLE
```

**轻触检测**（单击 = 左键，双击 = 右键）：

```typescript
const TAP_DISTANCE_THRESHOLD = 10;   // 像素
const TAP_TIME_THRESHOLD = 300;      // ms
const DOUBLE_TAP_THRESHOLD = 300;    // 两次轻触间隔

function onTouchEnd(e: TouchEvent) {
  const dx = lastX - startX;
  const dy = lastY - startY;
  const distance = Math.sqrt(dx * dx + dy * dy);
  const duration = Date.now() - tapStartTime;

  if (distance < TAP_DISTANCE_THRESHOLD && duration < TAP_TIME_THRESHOLD) {
    const now = Date.now();
    if (now - lastTapTime < DOUBLE_TAP_THRESHOLD) {
      sendRightClick();         // 双击 → 右键
      lastTapTime = 0;
    } else {
      pendingTapTimeout = setTimeout(() => {
        sendLeftClick();        // 单击 → 左键（延迟判定，等待可能的第二次点击）
      }, DOUBLE_TAP_THRESHOLD);
      lastTapTime = now;
    }
  }
}
```

**区分滚动与捏合**：

```typescript
function detectTwoFingerGesture(e: TouchEvent) {
  const t0 = e.touches[0], t1 = e.touches[1];

  // 计算质心移动（双指同向 = 滚动）
  const midX = (t0.clientX + t1.clientX) / 2;
  const midY = (t0.clientY + t1.clientY) / 2;
  const dx = midX - lastMidX;
  const dy = midY - lastMidY;

  // 计算指间距（变化 = 捏合）
  const dist = getDistance(t0, t1);
  const distDelta = dist - lastDist;

  if (Math.abs(distDelta) > PINCH_THRESHOLD) {
    sendPinch(distDelta);       // 捏合
  } else {
    sendScroll(dx, dy);         // 滚动
  }
}
```

### 5.3 触摸到光标的映射与加速

手机屏幕约 7cm 高，桌面约 50cm 宽。无加速需要滑动 7+ 次才能跨越屏幕；纯线性缩放则精确工作变得不可能。

**推荐的三段式分段加速**：

```typescript
function applyAcceleration(dx: number, dy: number): [number, number] {
  const speed = Math.sqrt(dx * dx + dy * dy);
  const dt = Date.now() - lastMoveTime;
  lastMoveTime = Date.now();
  const velocity = speed / Math.max(dt, 1);

  let multiplier: number;
  if (velocity < 0.5) {
    multiplier = 1.0;                                    // 精确模式
  } else if (velocity < 2.0) {
    multiplier = 1.0 + (velocity - 0.5) * 0.8;          // 中速线性斜坡
  } else {
    multiplier = 2.2 + Math.min((velocity - 2.0) * 0.5, 3.0);  // 高速区，上限 5.2x
  }

  return [dx * multiplier, dy * multiplier];
}
```

### 5.4 通信协议设计

#### WebSocket（推荐，主流选择）

**为什么选 WebSocket**：设置简单、无需信令服务器、浏览器普遍支持、局域网延迟足够。

**最优消息格式——二进制，而非 JSON**：

```typescript
// 二进制格式（移动消息仅 5 字节）
// Byte 0:    消息类型 (1=移动, 2=点击, 3=滚动, 4=捏合)
// Bytes 1-2: dx (Int16, 有符号, 小端)
// Bytes 3-4: dy (Int16, 有符号, 小端)

const ws = new WebSocket(url);
ws.binaryType = 'arraybuffer';

function sendMouseMove(dx: number, dy: number) {
  const buf = new ArrayBuffer(5);
  const view = new DataView(buf);
  view.setUint8(0, 1);          // type: move
  view.setInt16(1, dx, true);   // little-endian
  view.setInt16(3, dy, true);
  ws.send(buf);
}
```

**对比**：JSON `{"type":"move","dx":10,"dy":-5}` ≈ 30 字节 + UTF-8 开销。二进制 = 5 字节 + WebSocket 帧头 2-6 字节。在 60+ 次/秒的移动频率下，差异显著。

**关键性能优化**：
- 服务端 socket 设置 `TCP_NODELAY`（禁用 Nagle 算法）
- 使用 `ws.binaryType = 'arraybuffer'`
- 每次 `touchmove` 都发送，**不要**批量合并（局域网延迟比带宽更重要）
- 服务端如果同帧收到多条消息，只处理最新的

#### WebRTC DataChannel（更极致延迟的选项）

| 维度 | WebSocket | WebRTC DataChannel |
|------|-----------|-------------------|
| 协议 | TCP | SCTP over UDP |
| 局域网延迟 | ~5-10ms | ~1-3ms |
| 设置复杂度 | 简单（HTTP upgrade） | 复杂（ICE/STUN/TURN 信令） |
| 可靠性 | 保证有序 | 可配置（不可靠/无序可用） |
| 队头阻塞 | 是（TCP） | 否（无序模式下） |

**何时用 WebRTC**：需要亚 5ms 延迟或同时添加视频流（屏幕镜像 + 触控板）时。纯触控板在局域网下，WebSocket 的延迟差异不可感知。

## 六、对本项目的建议

基于以上研究，为"手机变无线触控板"项目提出以下建议：

### 6.1 必须实现的核心功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 单指移动 → 光标移动 | P0 | 增量模式，delta-based |
| 单指轻触 → 左键点击 | P0 | 距离 < 10px + 时间 < 300ms |
| 双指滚动 | P0 | 质心移动 + 指间距稳定 = 滚动 |
| 双指轻触 → 右键点击 | P0 | 两指同时轻触 |
| 指针加速曲线 | P0 | 三段式分段加速（精确/中速/高速） |
| 长按 → 拖拽 | P1 | 按住不动 300ms+ 后进入拖拽模式 |
| 双指捏合 → 缩放 | P1 | 指间距变化量映射到 Ctrl+滚轮 |

### 6.2 推荐技术方案

### 6.3 关键算法建议

1. **触摸预处理**：使用 **One Euro Filter** 平滑触摸数据，消除抖动同时保持响应性
2. **指针加速**：参考 macOS 的**分段线性查找表**或 Raw Accel 的 **Natural 曲线**，实现三段式加速
3. **惯性滚动**：使用**指数衰减模型**（`v = v₀ × 0.998^n`），配合橡皮筋边界效果
4. **手势识别**：实现有限状态机（`IDLE → TRACKING / MULTI_TOUCH → gesture detection`）
5. **防误触**：触摸面积 + 边缘区域 + 静止超时 三重判断
6. **通信协议**：WebSocket 二进制帧（5字节/消息），服务端设置 `TCP_NODELAY`

### 6.4 需要深入测试的体验细节

- [ ] 手指首次接触时的"拾取感"（不应有跳跃）
- [ ] 慢速移动时的精确度（亚像素累积 + 死区处理）
- [ ] 快速滑动后的减速是否自然（惯性滚动衰减率调优）
- [ ] 滚动到边界时的橡皮筋回弹感
- [ ] 从单指移动到双指滚动的切换流畅度
- [ ] 手掌误触率（特别是边缘区域）
- [ ] 不同手机屏幕尺寸下的灵敏度适配
