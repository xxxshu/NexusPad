## USB 连接问题解决方案

根据用户的诊断信息，问题已经分析清楚：

### 问题1: "仅充电"模式
- **现象**: `18D1:2D01 (无法打开) [AOA]`
- **原因**: Windows 缺少 WinUSB 驱动
- **解决方案**: 使用 Zadig 安装 WinUSB 驱动

### 问题2: "传输文件/Android Auto"模式  
- **现象**: `22D9:2765 (无法打开) [Android] → 打开失败: Access denied`
- **原因**: 设备被系统驱动占用
- **解决方案**: 切换 USB 模式到「仅充电」

## 使用步骤

1. **打开桌面端**，进入设置 → 通用 → USB 连接
2. **使用调试工具**:
   - 点击「驱动检测」查看当前状态
   - 点击「详细调试」生成日志
   - 点击「驱动修复指导」获取方案
3. **根据指导操作**:
   - 安装 WinUSB 驱动 或 切换 USB 模式
   - 重试连接

## 性能说明

- **AOA 连接**: 提供 <2ms 的低延迟，适合游戏手柄、陀螺仪
- **拒绝 ADB 转发**: 用户明确要求不使用 ADB 转发以避免性能损失
- **必须使用 AOA**: 为保证低延迟，必须通过 AOA 连接
