# NexusPad 开发计划

> 把手机变成电脑的无线触控板、键盘和游戏手柄 —— 局域网扫码即连。

---

## 一、项目介绍

### 1.1 产品定位

NexusPad 是一款将手机变为电脑外设的工具软件。用户通过扫码连接后，手机可以充当：

- **触控板** — 单指移动、双指滚动/缩放、点击/双击/长按拖动
- **键盘** — 实时输入、拼音输入法、功能键面板（Esc/Tab/Del/方向键/组合键）
- **游戏手柄**（规划中）— 虚拟 Xbox 360 / DualShock 4 手柄，通过 ViGEmBus 驱动

### 1.2 连接方式

- **局域网连接**（已实现）— 手机和电脑在同一网络下，扫码即连
- **USB 连接**（规划中）— 通过 ADB 端口转发，延迟更低更稳定

### 1.3 技术架构

```
手机端 (Flutter)                     PC 桌面端 (Tauri v2 / Rust)
┌──────────────────┐                ┌─────────────────────────────┐
│                  │   WebSocket    │  axum HTTP + WS Server      │
│  触控板手势引擎   │ ◀── JSON ───▶ │  ├─ 鼠标/键盘模拟 (enigo)   │
│  OSK 键盘        │   或二进制     │  ├─ 剪贴板 (arboard)        │
│  游戏手柄 UI     │                │  ├─ IME 状态管理 (平台 API)  │
│  PIN 认证        │                │  └─ ViGEmBus 手柄模拟 (新)  │
│                  │                │                             │
│  模式选择页       │                │  Tauri 桌面 GUI (React)     │
│   ├─ 触控板模式   │                │   ├─ 服务启停               │
│   └─ 手柄模式     │                │   ├─ QR 码 / PIN 显示      │
│                  │                │   ├─ 设置面板               │
│                  │                │   └─ ViGEmBus 驱动安装引导  │
└──────────────────┘                └─────────────────────────────┘
```

## 三、通信协议

### 3.1 传输层

WebSocket JSON，连接 `ws://<host>:8765/ws`

### 3.2 客户端 → 服务端 (ClientMsg)

| action (`a`) | 字段 | 说明 |
|---|---|---|
| `mv` | `x, y` | 鼠标移动（整数像素，增量值）— 最高频 60-120Hz |
| `clk` | `b` (0=左,1=中,2=右) | 点击 |
| `dbl` | — | 双击 |
| `md` | `b` | 鼠标按下（拖拽开始） |
| `mu` | `b` | 鼠标抬起（拖拽结束） |
| `scr` | `y, x` | 滚动 |
| `pz` | `m` | 缩放（>1放大，<1缩小） |
| `type` | `t` | 文本输入 |
| `key` | `k` | 按键按下 |
| `kp` | `k` | 按键（按下+释放） |
| `bs` | `n` | 退格（连续删除次数） |
| `ime` | `mode` | 切换IME（`"on"`/`"off"`） |
| `ime_toggle` | — | 按IME物理切换键 |
| `ime_refresh` | — | 刷新IME状态 |
| `auth` | `pin` | PIN认证 |
| `approval_resp` | `r` | 设备审批（`"accept"`/`"reject"`） |

### 3.3 服务端 → 客户端 (ServerMsg)

| action (`a`) | 字段 | 说明 |
|---|---|---|
| `ctrl_ok` | `proot?` | 连接成功，开始控制 |
| `wait` | `reason?` | 等待（被拒绝/超时/忙） |
| `approval_req` | `name, time, ip` | 设备审批请求 |
| `auth_required` | — | 需要PIN认证 |
| `auth_fail` | — | 认证失败 |
| `ime_init` | `status` ("EN"/"ZH") | IME初始状态 |
| `ime_state` | `status` | IME状态变化 |

## 四、接下来的开发计划

### Phase 1: Flutter 触控板模式（最小可用版本）

**目标**: 手机端能连上桌面端，控制鼠标移动和点击

- [ ] 项目配置：横屏锁定、全屏沉浸式、状态栏隐藏
- [ ] 模式选择页：进入 App 后选择"触控板键盘"或"游戏手柄"
- [ ] 连接流程：扫码页（调用相机扫桌面端 QR 码）→ 解析 URL → WebSocket 连接 → PIN 认证
- [ ] 触控板手势引擎（核心性能模块）：
  - 单指移动 → EMA 平滑 → 自适应加速 → 亚像素累加 → 整数截断 → 即时发送
  - 单击 / 双击 / 右键（双指点击）
  - 长按拖拽（400ms 延迟后进入 drag 状态）
  - 双指滚动 + 非线性曲线 + 惯性滚动
  - 双指缩放（pinch）
- [ ] 震动反馈（HapticFeedback API）
- [ ] 顶栏：连接状态、功能键/键盘切换按钮

### Phase 2: 键盘与拼音输入法

- [ ] OSK 键盘：QWERTY alpha + ?123 symbols 双层
- [ ] Shift 状态机（小写 → Shift → 锁定 → 小写）
- [ ] FK 功能键面板：Esc/Tab/Del/方向键/修饰键/组合键
- [ ] 拼音引擎：最长匹配 + 候选词栏 + 翻页
- [ ] 退格长按重复（400ms 启动，80ms 间隔）
- [ ] IME 切换（短按=服务端物理键，长按=本地 UI 翻转）

### Phase 3: 游戏手柄模式

- [ ] 虚拟手柄 UI：两个摇杆 + ABXY 按键 + 扳机 + 肩键（Flutter Canvas 自定义绘制）
- [ ] 手柄通信协议扩展：新增 `gamepad_state` 消息（摇杆轴 + 按键位掩码）
- [ ] 桌面端 ViGEmBus 集成：创建虚拟 Xbox 360 / DualShock 4 设备
- [ ] 驱动安装引导：桌面端检测 ViGEmBus 是否安装，提供下载链接和安装指引
- [ ] 摇杆优化：死区、响应曲线、亚像素平滑

### Phase 4: USB 连接 + 优化

- [ ] USB 连接方案：通过 ADB 端口转发，手机通过 USB 连接电脑
- [ ] 连接方式自动检测：USB 优先，回退到局域网
- [ ] 性能 profiling：触控延迟端到端测量，目标 < 15ms
- [ ] 异常处理：弱网、断连重连、后台保活

### Phase 5: 上线准备

- [ ] 多机型测试（高/中/低端 Android，iPhone）
- [ ] 桌面端安装引导优化（ViGEmBus 静默安装选项）
- [ ] 打包发布：Android APK / iOS TestFlight
- [ ] README 更新

## 五、关键技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 手机端框架 | Flutter | 一套代码出 Android+iOS，自定义绘制性能好，UDP 原生支持 |
| PC 端框架 | Tauri v2 + Rust | 已有实现，跨平台，包体小 |
| 通信协议 | WebSocket JSON | 已有实现，兼容现有网页版 |
| 手势引擎位置 | 手机端本地计算 | 避免渲染延迟，即时响应 |
| 虚拟手柄驱动 | ViGEmBus | 成熟方案，支持 Xbox 360 和 DS4，游戏兼容性好 |
| 微信小程序 | ❌ 已放弃 | WebSocket 必须连已备案 HTTPS 域名，无法直连局域网 IP |
